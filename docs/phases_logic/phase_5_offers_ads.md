# Phase 5: Offers & Ads (العروض والإعلانات)

## الهدف والمغزى
هذا هو صلب العمل التجاري للتطبيق. السماح للشركات ومقدمي الخدمات بنشر عروضهم، والسماح للمؤثرين والشركات بنشر إعلاناتهم مع إمكانية تتبع مدى وصول هذه الإعلانات (الإحصائيات).

## الملفات المرتبطة
- **Validators**: `src/validators/offer.validator.ts`, `src/validators/ad.validator.ts`
- **Controllers**: `src/controllers/offer.controller.ts`, `src/controllers/ad.controller.ts`
- **Services**: `src/services/offer.service.ts`, `src/services/ad.service.ts`
- **Routes**: `src/routes/offer.routes.ts`, `src/routes/ad.routes.ts`
- **Database**: `supabase/migrations/004_create_offers_and_ads.sql`

## اللوجيك (كيف تعمل)
### 1. العروض (Offers):
- **الإنشاء والإدارة**: `POST /api/v1/offers` يتيح للمستخدم نشر عرض مع تحديد العنوان، الوصف، السعر، والتصنيف.
- **التصفح والفلترة**: `GET /api/v1/offers` هي النقطة الأهم، حيث تستقبل Query Parameters مثل `categoryId`, `minPrice`, `maxPrice`، وتستخدم الـ Service هذه المعاملات لبناء استعلام (Query) معقد يجلب العروض المطابقة للبحث، مع دعم تقسيم الصفحات (Pagination).
- **المشاهدات (Views)**: عندما يطلب التطبيق تفاصيل عرض معين `GET /api/v1/offers/:id`، يقوم الـ Service بإرسال الطلب لجلب البيانات، وفي نفس الوقت يطلب من قاعدة البيانات زيادة حقل `views_count` بمقدار 1.

### 2. الإعلانات (Ads):
- **الإنشاء**: مثل العروض ولكن بحقول مختلفة (مثل `target_url`، `call_to_action`).
- **التتبع (Tracking)**:
  - الإعلانات تحتاج لإحصائيات دقيقة. بدلاً من تحديث الإعلان مع كل مشاهدة وجعله بطيئاً، قمنا بعمل نقاط نهاية خفيفة جداً:
  - `POST /api/v1/ads/:id/impression`: يطلبها التطبيق بصمت (في الخلفية) عندما يظهر الإعلان على شاشة المستخدم، فيزيد `impressions_count`.
  - `POST /api/v1/ads/:id/click`: يطلبها التطبيق عندما يضغط المستخدم على الإعلان، فيزيد `clicks_count`.
- **الإحصائيات**: `GET /api/v1/ads/:id/stats` يقوم بحساب معدل النقر للظهور (CTR) عبر قسمة النقرات على الانطباعات وإرجاعها للوحة تحكم صاحب الإعلان.

## الـ User Flow (رحلة المستخدم في التطبيق)
- **كمعلن**: يدخل لصفحة إضافة إعلان، يرفع صورة، يضع رابط، ويضغط حفظ. لاحقاً يدخل لصفحة الإحصائيات ليرى كم شخص شاهد وضغط على إعلانه (كل هذا يتم عبر نقاط الـ Ads).
- **كمستخدم عادي**: يتصفح العروض، التطبيق يستخدم نقطة نهاية `GET /offers` ويظهر العروض. عندما يضغط على عرض معين، يتم استدعاء `GET /offers/:id` ليرى التفاصيل وتُسجل مشاهدة.

## كيفية الصيانة والتعديل (Maintenance)
- **إضافة حقل لفلتر جديد (مثلاً البحث بالكلمات)**: تحتاج لإضافته كـ Query Parameter في `offer.validator.ts`، ثم استلامه في `offer.controller.ts` وتمريره لـ `offer.service.ts`. داخل الـ Service، تضيف شرط `ilike` جديد للاستعلام في Supabase.
- **فصل المشاهدات لمنع التكرار**: حالياً، يتم زيادة المشاهدة في كل مرة يُفتح العرض. إذا أردت جعلها "مشاهدة فريدة لكل مستخدم"، ستحتاج لإنشاء جدول جديد `offer_views` يسجل من شاهد ماذا ومتى، وتعديل الـ Service ليتحقق قبل الزيادة.

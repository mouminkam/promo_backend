# Phase 3: Profiles (الملفات الشخصية)

## الهدف والمغزى
إدارة بيانات المستخدمين الإضافية التي لا تخص المصادقة (مثل الاسم، البايو، الصور الشخصية، النوع، الموقع). فصل الـ Auth عن الـ Profiles هو أفضل ممارسة، حيث يتكفل Supabase Auth بالأمان، ويتكفل جدول `profiles` بالبيانات التي ستظهر للعامة.

## الملفات المرتبطة
- **Validators**: `src/validators/profile.validator.ts`
- **Controllers**: `src/controllers/profile.controller.ts`
- **Services**: `src/services/profile.service.ts`
- **Routes**: `src/routes/profile.routes.ts`
- **Database**: `supabase/migrations/001_create_profiles.sql`

## اللوجيك (كيف تعمل)
1. **الإنشاء (التلقائي)**: لا يوجد Endpoint لـ "إنشاء ملف شخصي". بدلاً من ذلك، عند التسجيل في Phase 2، يعمل Database Trigger لينسخ بيانات المستخدم (الـ ID، الإيميل، الاسم) وينشئ صفاً في جدول `profiles`.
2. **الاستعلام والتعديل**:
   - `GET /api/v1/profiles/me`: الـ Service يستخدم الـ ID الموجود في الـ Token (الذي وفره الـ `auth.middleware.ts`) لجلب بيانات المستخدم الخاصة من قاعدة البيانات.
   - `PUT /api/v1/profiles/me`: يقوم بتحديث الحقول المسموح بها (مثل الـ Bio، الـ Username، الـ Website).
3. **الملفات العامة (Public Profiles)**:
   - `GET /api/v1/profiles/:id`: جلب بيانات شخص آخر لكي تظهر في صفحته. الـ RLS في قاعدة البيانات تسمح بقراءة هذه البيانات للعامة لأنها "عامة".
4. **التوثيق (Verification)**:
   - المستخدم يرسل طلب `POST /api/v1/profiles/me/verify-request` لتأكيد حسابه، وهذا قد يتطلب إرفاق أوراق (يعتمد على تصميم الواجهة لاحقاً)، لكنه يغير حالة الطلب فقط بانتظار موافقة الـ Admin.

## الـ User Flow (رحلة المستخدم في التطبيق)
- يدخل المستخدم التطبيق لأول مرة بعد التسجيل.
- يذهب لصفحة "حسابي".
- يرسل التطبيق طلب `GET /me` ويجلب بياناته ليعرضها فارغة جزئياً.
- يملأ المستخدم الـ Bio ويغير اسمه، يضغط "حفظ".
- يرسل التطبيق `PUT /me` بالبيانات الجديدة. السيرفر يحدثها ويعيد رسالة نجاح.
- عندما يضغط المستخدم على اسم شخص آخر في التطبيق، يرسل التطبيق `GET /profiles/1234` لعرض صفحته العامة.

## كيفية الصيانة والتعديل (Maintenance)
- **إضافة حقل جديد للملف الشخصي (مثلاً: رابط انستغرام)**:
  1. أنشئ Migration جديد يضيف العمود `instagram_url` لجدول `profiles`.
  2. أضفه في `profile.validator.ts` ليتمكن المستخدم من إرساله في الـ PUT.
  3. سيقوم الـ Service بتحديثه تلقائياً لأننا نمرر البيانات الصالحة مباشرة لـ Supabase.
